  ⚡ 本地存储不工作？按这个顺序检查！
═══════════════════════════════════════════════════════════════

第1步：运行诊断测试 🔍
───────────────────────────────────────────────────────────────
□ 双击打开"测试本地存储.html"
□ 点击"运行测试"按钮
□ 如果第1个测试失败 → 跳到第2步
□ 如果第1个测试成功 → 跳到第3步

第2步：检查浏览器设置 ⚙️
───────────────────────────────────────────────────────────────
□ 检查是否使用隐私/无痕模式
  → 如果是，关闭并使用普通窗口

□ Chrome用户：
  设置 → 隐私和安全 → Cookie → 允许所有Cookie

□ Safari用户：
  偏好设置 → 隐私 → 取消"阻止所有Cookie"

□ Firefox用户：
  设置 → 隐私与安全 → 标准或宽松

□ 检查浏览器版本
  → 如果是旧版本，更新到最新版

第3步：测试主应用 📱
───────────────────────────────────────────────────────────────
□ 打开 src/index.html
□ 按 F12 打开开发者工具
□ 切换到 Console 标签
□ 导入一个Excel文件
□ 查看是否显示："✅ 数据已保存到localStorage"

如果没有这条消息：
  → 检查是否有红色错误信息
  → 截图发送求助

如果有这条消息：
  → 继续下一步

第4步：验证数据保存 💾
───────────────────────────────────────────────────────────────
□ 在开发者工具中，切换到 Application 标签
  （Firefox是Storage标签）
□ 左侧找到 Local Storage → file://
□ 查看是否有 petEnglishData 这一项
□ 点击查看内容是否有数据

如果没有这项或内容为空：
  → 保存失败，查看Console的错误
  → 可能是存储空间不足

如果有数据：
  → 继续下一步

第5步：测试恢复 🔄
───────────────────────────────────────────────────────────────
□ 查看底部统计的题目数量（记住这个数字）
□ 完全关闭浏览器（不是标签，是整个浏览器）
□ 重新打开 src/index.html
□ 再次查看底部统计
□ 数字应该和之前一样

如果数字变回0：
  → 说明浏览器在退出时清除了数据
  → 检查浏览器设置："退出时清除数据"

  🎯 快速诊断决策树
═══════════════════════════════════════════════════════════════

测试页面第1个测试失败？
  └─ 是 → 浏览器设置问题
      └─ 使用无痕模式？
          ├─ 是 → 换普通模式
          └─ 否 → 检查Cookie设置
  └─ 否 → 继续

Console显示"已保存"？
  └─ 是 → 继续
  └─ 否 → Excel导入有问题
      └─ 检查文件格式
      └─ 查看错误信息

Application标签有数据？
  └─ 是 → 继续
  └─ 否 → 保存失败
      └─ 存储空间不足？
      └─ 安全策略阻止？

关闭重开数据还在？
  └─ 是 → ✅ 问题解决！
  └─ 否 → 浏览器退出时清除
      └─ 检查"退出时清除数据"设置
