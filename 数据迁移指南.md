# 📦 数据迁移指南

## 问题说明

打包后的应用使用独立的 localStorage，开发时的打卡记录不会自动迁移。

## 解决方案

### 方法一：从开发版本导出并导入（推荐）

如果您还能在浏览器中打开开发版本：

#### 步骤 1：在开发版本中导出打卡记录

1. 在浏览器中打开开发版本（`src/index.html`）
2. 点击 **"📅 打卡记录"** 按钮
3. 点击 **"💾 导出打卡记录"** 按钮
4. 保存下载的 JSON 文件（例如：`checkin-records.json`）

#### 步骤 2：在打包后的应用中导入

1. 打开打包后的应用（PET魔法英语屋）
2. 点击 **"📅 打卡记录"** 按钮
3. 点击 **"📥 导入打卡记录"** 按钮
4. 选择刚才导出的 JSON 文件
5. 完成！打卡记录已恢复

---

### 方法二：直接访问浏览器 localStorage（高级）

如果您熟悉浏览器开发者工具：

1. 在浏览器中打开开发版本
2. 按 `F12` 打开开发者工具
3. 切换到 **"Application"** 标签（或 **"存储"** 标签）
4. 在左侧找到 **"Local Storage"** → 点击您的网站地址
5. 找到 `petCheckinRecords` 键，复制其值
6. 在打包后的应用中：
   - 按 `Cmd+Option+I`（Mac）或 `Ctrl+Shift+I`（Windows）打开开发者工具
   - 在 Console 中运行：
   ```javascript
   localStorage.setItem('petCheckinRecords', '粘贴刚才复制的值');
   location.reload();
   ```

---

### 方法三：重新开始

如果无法访问开发版本的数据，只能重新开始记录。新的打卡记录会自动保存。

---

## 预防措施

为了避免将来再次丢失数据，建议：

1. **定期导出备份**：每周或每月导出一次打卡记录
2. **保存备份文件**：将导出的 JSON 文件保存到安全的位置（如 iCloud、OneDrive 等）

---

## 注意事项

- 导入时会自动去重（基于时间戳）
- 导入后会自动重新计算星星和积分
- 导入不会覆盖现有数据，只会合并新记录

---

## 需要帮助？

如果遇到问题，请检查：
1. JSON 文件格式是否正确
2. 文件是否包含有效的打卡记录数据
3. 浏览器控制台是否有错误信息
